Timestamp Microservice Challenge

User stories:
    a) I can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2016).
    b) If it does, it returns both the Unix timestamp and the natural language form of that date.
    c) If it does not contain a date or Unix timestamp, it returns null for those properties.

Example usage:

https://fierce-anchorage-54948.herokuapp.com/January%201,%202016
https://fierce-anchorage-54948.herokuapp.com/1451606400

Example output:

{ "unix": 1451606400, "natural": "January 1, 2016" } {"unix":null,"natural":null}
